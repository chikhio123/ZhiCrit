# 变更审查

改完代码后必须执行，不跳步。

## 三步走

### 1. 抓变动面
列出本次改了什么——字段、函数、条件、入参、返回值。

### 2. 追影响链
对每个变动点：
- **数据流**：谁写、谁读、结构变了读者崩不崩
- **控制流**：枚举所有分支组合，每个组合走到哪
- **调用链**：搜引用，确认没残留旧调用、新代码有调用方

### 3. 输出审查表
用表格列场景 → 输入 → 路径 → 结果，覆盖全部组合。确认无遗漏后说"审查通过"。

## 常见坑（逐项过一遍）

- 不同模式/不同 depth/reuse 结果 的组合走完了吗
- 状态在 reset/clear/重分析 时丢不丢
- 缓存/复用 跨文章泄漏？不同 mode 串了？
- 异步：await 前状态变了怎么办
- UI 端：进度条/按钮/v-html/disabled 受不受影响
